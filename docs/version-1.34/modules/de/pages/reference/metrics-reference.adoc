== Policy-Server

Die Komponente Policy-Server initialisiert und führt Richtlinien aus.
Beim Empfang von Anfragen vom Kubernetes-API-Server leitet er die Anfrage an die Richtlinie weiter und gibt die von der Richtlinie bereitgestellte Antwort an den Kubernetes-API-Server zurück.

=== Metriken

[NOTE]
====
https://opentelemetry.io/docs/concepts/signals/baggage/[Baggage] Schlüsselwertattribute werden der Metrik hinzugefügt, um zusätzliche Informationen bereitzustellen.
====


|===
| Name | Typ | Baggage

| `kubewarden_policy_evaluations_total`
| +++Counter
| <<_kubewarden_policy_evaluations_total,Baggage>>
|===

==== `kubewarden_policy_evaluations_total`

===== Baggage

|===
| Kennung | Beschreibung

| `policy_name`
| Name der Richtlinie

| `resource_name`
| Name der bewerteten Ressource

| `resource_kind`
| Art der bewerteten Ressource

| `resource_namespace`
| Namespace der bewerteten Ressource. Nicht vorhanden, wenn die Ressource clusterweit ist.

| `resource_request_operation`
| Operationstyp: `CREATE`, `UPDATE`, `DELETE`, `PATCH`, `WATCH`...

| `accepted`
| Wurde die Anfrage akzeptiert?

| `mutated`
| Wurde die Anfrage verändert?

| `error_code`
| Fehlercode, der von der Richtlinie zurückgegeben wird, falls sie abgelehnt wird, falls vorhanden.
  Nicht vorhanden, wenn die Richtlinie keinen bereitgestellt hat.
|===
