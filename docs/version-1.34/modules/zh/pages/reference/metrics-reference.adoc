== 策略服务器

策略服务器组件初始化并运行策略。
在接收到来自Kubernetes API服务器的请求时，它将请求转发给策略，并将策略提供的响应返回给Kubernetes API服务器。

=== 指标

[NOTE]
====
https://opentelemetry.io/docs/concepts/signals/baggage/[Baggage]键值属性被添加到指标中，以提供额外信息。
====


|===
| 名称 | 类型 | Baggage

| `kubewarden_policy_evaluations_total`
| 计数器
| <<_kubewarden_policy_evaluations_total,Baggage>>
|===

==== `kubewarden_policy_evaluations_total`

===== Baggage

|===
| 标签 | 说明

| `policy_name`
| 策略名称

| `resource_name`
| 被评估资源的名称

| `resource_kind`
| 被评估资源的类型

| `resource_namespace`
| 被评估资源的名称空间。如果资源是集群范围的，则不存在。

| `resource_request_operation`
| 操作类型：`CREATE`，`UPDATE`，`DELETE`，`PATCH`，`WATCH`...

| `accepted`
| 请求是否被接受？

| `mutated`
| 请求是否被修改？

| `error_code`
| 如果被拒绝，策略返回的错误代码（如果有的话）。
  如果策略没有提供，则不显示。
|===
