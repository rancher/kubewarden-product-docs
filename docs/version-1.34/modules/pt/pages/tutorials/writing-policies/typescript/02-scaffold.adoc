= Criando uma nova política de validação
:revdate: 2026-03-18
:page-revdate: {revdate}
:description: Crie uma nova política de validação para rejeitar nomes de host inválidos em objetos Pod por meio da configuração das definições da política, testada com as ferramentas fornecidas.
:keywords: {project-name}, kubewarden, kubernetes, escrevendo políticas em TypeScript, nova política de validação
:doc-type: tutorial
:doc-topic: {project-name}, kubewarden, escrevendo políticas, TypeScript, criando uma nova política de validação
:doc-persona: kubewarden-policy-developer
:current-version: {page-origin-branch}

Este tutorial abrange a criação de uma política que valida os nomes de host dos objetos Pod.

A política é rejeitar todos os Pods que usam um ou mais nomes de host na lista de negação.
Você fornece a configuração da política usando as configurações de runtime.

Para resumir, as configurações da política devem ser assim:

[source,yaml]
----
denied_hostnames:
  - bad-host
  - forbidden-host
----

A política rejeita a criação deste Pod:

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: bad-host
  containers:
    - name: nginx
      image: nginx:latest
----

No entanto, aceita a criação deste Pod:

[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  hostname: allowed-host
  containers:
    - name: nginx
      image: nginx:latest
----

== Estruturando um novo projeto de política

Você pode criar um novo projeto de política usando o https://github.com/kubewarden/js-policy-template[repositório de modelo]. Selecione o botão verde "Usar este modelo" próximo ao topo da página e siga o assistente do GitHub.
Clone o repositório localmente e atualize o arquivo `package.json` para refletir os detalhes da sua política:

[source,json]
----
{
  "name": "your-policy-name",
  "version": "1.0.0",
  "description": "Your policy description",
  "repository": {
    "type": "git",
    "url": "https://github.com/your-username/your-policy-name"
  }
}
----

Certifique-se de usar um caminho de repositório que corresponda ao seu repositório real do GitHub.
