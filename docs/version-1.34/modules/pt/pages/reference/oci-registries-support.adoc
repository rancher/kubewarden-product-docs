Você distribui suas políticas de {project-name} como
https://opencontainers.org[Artefatos OCI]
usando registros regulares da Open Container Initiative (OCI).

{short-project-name} O armazenamento da política está ao lado das imagens de contêiner.
Elas não requerem configuração ou manutenção extra
além da necessária para imagens de contêiner regulares.

[NOTE]
====
Você pode adicionar um registro que funcione com {short-project-name} ou
corrigir quaisquer imprecisões do registro usando o
botão "Editar esta página" na parte inferior da página
====


== Projetos que implementam registros OCI

* https://goharbor.io/[Harbor].
* https://github.com/distribution/distribution[Distribution] (https://github.com/distribution/distribution/releases/tag/v2.7.0[≥ 2.7.0]).

== Problemas conhecidos

=== Docker Hub

Atualmente, o Docker Hub não suporta artefatos OCI, portanto não pode ser usado para armazenar {short-project-name} políticas.
A Docker Inc. anunciou que o Docker Hub suportará artefatos OCI no https://www.docker.com/blog/announcing-docker-hub-oci-artifacts-support/[futuro].

=== JFrog

Embora o JFrog suporte artefatos OCI, é apenas parcialmente possível enviar para ele, ao seguir sua especificação. https://github.com/kubewarden/kwctl/issues/59[Leia mais aqui].
